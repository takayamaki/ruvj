# /log-progress

ストーリー完了・PR作成のタイミングで当日の作業をNotionに記録するスキル。

## 事前チェック

1. `git-agent log --oneline -30` と `git-agent status` で当日の作業範囲を把握する。
   どのbranchで何をcommitしたか、マージ状態、pushの有無を確認する。
2. 既に今日分のページが存在する場合は **追記** する。新規作成ではない。

## 書き込み先と粒度

| 書き込み先 | parent page_id | サブページ名 | 粒度 |
|---|---|---|---|
| 開発日誌 | `33b34f9c84e981fea796f02b85db2578` | `YYYY-MM`（月次） | 詳細。やったこと・検討案・**却下案と理由**・技術的な学び・プロセスの学び・次にやること |

開発日誌の月次ページ内には `## YYYY-MM-DD: タイトル` 見出しで当日セクションを追加する。

## 書き込み時の注意（hookで保護されている）

Notionのtoolに `\n` リテラル（バックスラッシュ+n の2文字）を渡すと文字"n"になり全文が見出し化する。
**必ず実改行 (0x0A) を直接埋め込む**こと。`content_updates[].new_str` / `pages[].content` 全てで同じ。
違反は `.claude/tools/notion/newline-guard.ts` が PreToolUse hook で exit 2 停止する。

## 手順

1. **既存ページの取得**
   - `notion-fetch` で開発日誌の `YYYY-MM` サブページとセッションログの `MM-DD` サブページを探す。
   - 当月/当日のページが無ければ `notion-create-pages` で作成。
2. **開発日誌への追記**
   - 当日セクション `## YYYY-MM-DD: タイトル` を月次ページ末尾に `notion-update-page` の `append` で追加。
   - やったこと（実装項目の箇条書き）・検討案・却下理由・学び・次にやることを含める。
3. **将来タスクの起票**
   - 会話中に出た「別件だけど〜したい」「〜するときに忘れずメモ」系の発言を拾い、
     Notion Task DB（見つからなければユーザーに確認）に Phase/blocked by 付きで起票。
   - 起票した項目は箇条書きでユーザーに提示する。
4. **要約をユーザーに返す**
   - 書き込み先ページのURL、追加した見出し、起票タスク一覧を短くまとめて返す。

## やらないこと

- 開発日誌に書いた却下案と理由をセッションログにコピーしない。
- 当日以外のコミット/作業を勝手に書き込まない。
- Task DB のPhase/優先度を勝手に設定しない（既定値か、明らかなものだけ）。
